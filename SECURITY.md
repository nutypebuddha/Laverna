# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Laverna, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, email: **nutypebuddha@proton.me**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response

- Acknowledgment within 48 hours
- Assessment and fix timeline within 1 week
- Credit in release notes (unless you prefer anonymity)

## Scope

- Core reasoning engine (determinism guarantees)
- MCP server (if enabled)
- Build pipeline (supply chain)

## Out of Scope

- Third-party dependencies (report to upstream)
- Physical security of deployment environment

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest main | Yes |
| Previous commits | Best effort |
